# 底层驱动验证清单
> 每写完一个底层文件，按此清单写测试代码验证，确认现象正确再继续写下一个。

---

## 1. init.c — 系统初始化

**测试方法：** 只调用 `System_Init()`，不做任何其他操作，直接上电。

```c
void main()
{     
     System_Init();
}
```

| 操作 | 预期现象 | 说明 |
|------|----------|------|
| 直接上电 | **8个LED全灭** | P0=0xff 写入 LED 锁存器 |
| 直接上电 | **蜂鸣器不响、继电器不吸合** | P0=0x00 写入继电器锁存器 |

> ❌ 若上电后 LED 乱亮或蜂鸣器响，说明 init.c 有问题。

## 2. led.c — LED 控制

**测试方法：** 依次点亮不同 LED，观察是否与代码对应。

```c
pdata unsigned char ucled[8] = {0,0,0,0,0,0,0,0};

void main()
{
    System_Init();

    // 测试1：只亮 L1
    ucled[0] = 1;
    Led_Disp(ucled);
    Delay(50000);

    // 测试2：全亮
    unsigned char i;
    for(i=0;i<8;i++) ucled[i]=1;
    Led_Disp(ucled);
    Delay(50000);

    // 测试3：全灭
    led_off();
    while(1);
}
```

| 操作 | 预期现象 | 说明 |
|------|----------|------|
| ucled[0]=1，其余=0 | **只有 L1 亮** | 验证位映射正确 |
| ucled[7]=1，其余=0 | **只有 L8 亮** | 验证最高位正确 |
| 全部置1 | **L1~L8 全亮** | 验证合并字节正确 |
| 调用 led_off() | **全部熄灭** | 验证关灯函数正确 |

> ❌ 若亮的 LED 编号与 ucled 下标不对应，检查移位方向。

## 3. seg.c — 数码管显示

**测试方法：** 用 Timer1 中断扫描，在 Seg_Buf 里填数字，观察显示内容。

```c
idata unsigned char Seg_Pos = 0;
pdata unsigned char Seg_Buf[8] = {0,1,2,3,4,5,6,7}; // 显示 01234567

void Timer1_Isr(void) interrupt 3
{
    Seg_Pos = (Seg_Pos + 1) % 8;
    Seg_Disp(Seg_Pos, Seg_Buf[Seg_Pos], 0);
}

void main()
{
    System_Init();
    Timer1_Init();
    while(1);
}
```

| 填入 Seg_Buf 的值 | 预期现象 | 说明 |
|-------------------|----------|------|
| `{0,1,2,3,4,5,6,7}` | 显示 **01234567** | 验证数字 0~7 正确 |
| `{8,9,10,10,10,10,10,10}` | 显示 **89______**（后6位熄灭）| 验证数字 8、9 和熄灭码 |
| `{11,12,13,14,10,10,10,10}` | 显示 **CNLP____** | 验证字符 C/N/L/P |
| `{0+',', 10,10,10,10,10,10,10}` | 第1位显示 **0.**（带小数点）| 验证小数点逻辑（值>20）|

> ❌ 若数字显示错误，检查 seg_dula[] 数组顺序和大小写。
> ❌ 若只有1位亮，检查位选 `0x01 << wela` 和锁存时序。
> ❌ 若 C/N/L/P 乱码，检查 seg_dula[] 是否有15个元素。

## 4. key.c — 矩阵按键

**测试方法：** 读取按键值显示在数码管最后两位。

```c
void main()
{
    System_Init();
    Timer1_Init();
    while(1)
    {
        unsigned char k = Key_Read();
        if(k != 0)
        {
            Seg_Buf[6] = k / 10;
            Seg_Buf[7] = k % 10;
        }
    }
}
```

| 按键操作 | 预期数码管显示 | 说明 |
|----------|----------------|------|
| 按 S4 | **04** | |
| 按 S5 | **05** | |
| 按 S6 | **06** | |
| 按 S7 | **07** | |
| 按 S8 | **08** | |
| 按 S9 | **09** | |
| 同时按 S8+S9 | **89** | 验证联合按键 |
| 松开所有键 | 数值**保持不变** | 无键返回0，不刷新 |

> ❌ 若按 S4 显示的不是4，检查行列扫描顺序。
> ❌ 若同时按 S8+S9 只显示 8 或 9，检查联判是否在单键判断之前。

## 5. iic.c（ADC）— PCF8591 模数转换

**测试方法：** 读取通道3（滑动变阻器），将结果显示在数码管。

```c
void main()
{
    System_Init();
    Timer1_Init();
    while(1)
    {
        unsigned char adc = Ad_Read(0x03);
        Seg_Buf[5] = 10;             // 空
        Seg_Buf[6] = adc / 10 % 10;
        Seg_Buf[7] = adc % 10;
        Delay(10000);
    }
}
```

| 操作 | 预期现象 | 说明 |
|------|----------|------|
| 滑动变阻器拧到最小 | 数码管显示接近 **0** | 0V ≈ 0 |
| 滑动变阻器拧到最大 | 数码管显示接近 **255** | 5V ≈ 255 |
| 缓慢旋转 | 数值**平滑变化** | 无跳变说明I2C通信正常 |

> ❌ 若显示始终为 0 或 255 不变，检查 I2CStop 是否有括号、sda/scl 是否小写。
> ❌ 若数值跳动剧烈，检查 I2C_Delay 参数。

## 6. iic.c（EEPROM）— AT24C02 读写

**测试方法：** 写入数据，断电重启后读出，验证是否一致。

```c
unsigned char buf_w[3] = {0x11, 0x22, 0x33};
unsigned char buf_r[3] = {0,0,0};

void main()
{
    System_Init();
    Timer1_Init();

    EEPROM_Write(buf_w, 0x00, 3); // 写入地址0起的3个字节

    Delay(50000); // 等待写入完成

    EEPROM_Read(buf_r, 0x00, 3);  // 读回

    // 显示第一个读回的字节
    Seg_Buf[6] = buf_r[0] / 16;   // 高4位（十六进制）
    Seg_Buf[7] = buf_r[0] % 16;

    while(1);
}
```

| 操作 | 预期现象 | 说明 |
|------|----------|------|
| 写入后立即读回 | buf_r[0]=0x11, [1]=0x22, [2]=0x33 | 验证写读流程 |
| 断电重启只执行读操作 | 读回数据与上次写入**相同** | 验证掉电保存 |

> ❌ 若读回全为 0xFF，检查 EEPROM_Read 里是否用了 I2CReceiveByte（不是 I2CSendByte）。
> ❌ 若断电后数据丢失，检查写完后的等待延时是否足够。

## 7. onewire.c — DS18B20 温度传感器

**测试方法：** 读取温度显示在数码管。

```c
void main()
{
    System_Init();
    Timer1_Init();
    while(1)
    {
        unsigned char wendu = (unsigned char)rd_temperature();
        Seg_Buf[5] = 10;
        Seg_Buf[6] = wendu / 10 % 10;
        Seg_Buf[7] = wendu % 10;
        Delay(100000);
    }
}
```

| 操作 | 预期现象 | 说明 |
|------|----------|------|
| 正常放置 | 显示 **20~30** 之间的数值（室温）| 验证基本通信 |
| 用手捂住传感器 | 数值**缓慢上升** | 验证温度变化响应 |
| 用冰/凉水接触 | 数值**下降** | 验证方向正确 |

> ❌ 若显示 0 或 85，说明初始化失败，检查 DQ 引脚定义（P1^4）和复位时序。
> ❌ 若显示 128 等大数，说明读取了负温度，检查 high 字节处理。

## 8. Ultrasound.c — 超声波测距

**测试方法：** 连续测距，结果显示在数码管。

```c
void main()
{
    System_Init();
    Timer1_Init();
    while(1)
    {
        unsigned char dist = Ultra_Wave_Data();
        Seg_Buf[4] = 10;
        Seg_Buf[5] = dist / 100 % 10;
        Seg_Buf[6] = dist / 10  % 10;
        Seg_Buf[7] = dist % 10;
        Delay(100000);
    }
}
```

| 操作 | 预期现象 | 说明 |
|------|----------|------|
| 正前方放一本书，距离约 20cm | 显示接近 **20** | 验证基本测距 |
| 将书移远到约 40cm | 显示接近 **40** | 验证线性关系 |
| 无遮挡物（超出量程）| 显示 **0** | 验证超时返回0 |

> ❌ 若显示始终为 0，检查 Ultra_Wave_Init() 是否在 CR=1 前调用、US_TX 引脚是否为 P1^0。
> ❌ 若数值乱跳，检查 while 后面是否有分号（等待回波的空循环）。

## 9. Timer1 + 调度器框架

**测试方法：** 让 LED1 每 500ms 翻转一次，验证调度器时序。

```c
pdata unsigned char ucled[8] = {0,0,0,0,0,0,0,0};

void Led_Toggle()
{
    ucled[0] ^= 1;
    Led_Disp(ucled);
}

idata task_t Scheduler_Task[] =
{
    { Led_Toggle, 500, 0 },
};

void main()
{
    System_Init();
    Timer1_Init();
    Scheduler_Init();
    while(1)
        Scheduler_Run();
}
```

| 操作 | 预期现象 | 说明 |
|------|----------|------|
| 上电运行 | L1 **每隔 500ms 闪烁一次** | 验证 Timer1 1ms 中断正确 |
| 改 rate_ms 为 1000 | L1 **每隔 1s 闪烁一次** | 验证调度周期可调 |
| 改 rate_ms 为 100 | L1 **快速闪烁（肉眼看近似常亮）**| 验证短周期任务 |

> ❌ 若 L1 不闪烁，检查 main() 里是否调用了 Timer1_Init() / Scheduler_Init() / Scheduler_Run()。
> ❌ 若闪烁频率明显不对，检查 AUXR 是 &=0xBF 还是 |=0x40，以及 TL1/TH1 初值。
> ❌ 若程序跑飞（复位），检查 task_func() 调用是否有括号。
