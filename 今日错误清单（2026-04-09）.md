# 今日错误清单（2026-04-09）

> 内容来源：底层驱动代码练习，含两次练习（底层 / 新建文件夹）中发现的全部错误。  
> 每条包含：错误代码 → 正确代码 → 原因解释。

## 一、led.c

### 错误1：`temp_1_old` 初始值设错，上电后 LED 无反应

**错误代码：**
```c
unsigned char temp_1_old = 0xff;
```

**正确代码：**
```c
idata unsigned char temp_1_old = 0x00;
```

**原因：**  
`temp_1_old` 的语义是"上次写入硬件的 temp_1 值"。  
`system_init()` 将 P0=0xff 送入 LED 锁存器，对应 `temp_1=0x00`（全灭）。  
若 `ucled={1,1,1,1,1,1,1,1}`，计算出 `temp_1=0xFF`，与 `temp_1_old=0xFF` 相等，  
`if(temp_1 != temp_1_old)` 判断为假，LED 刷新代码永远不执行，上电后 LED 无反应。  
初始化为 `0x00` 才能与硬件初始状态一致，保证第一次调用能正常刷新。

---

### 错误2：变量未加 `idata` 存储类型

**错误代码：**
```c
unsigned char temp_1 = 0x00;
unsigned char temp_1_old = 0x00;
unsigned char temp_2 = 0x00;
unsigned char temp_2_old = 0xff;
```

**正确代码：**
```c
idata unsigned char temp_1 = 0x00;
idata unsigned char temp_1_old = 0x00;
idata unsigned char temp_2 = 0x00;
idata unsigned char temp_2_old = 0xff;
```

**原因：**  
`temp_1`/`temp_1_old` 在中断和主循环中频繁访问。`idata` 将变量存入 8051 内部 RAM，  
访问速度最快。不加存储类型默认放在外部 RAM，访问慢，竞赛代码应尽量用 `idata`。

---

### 错误3：缺少 `Led_Off()`、`Motor()`、`Relay()` 函数

**原因：**  
- `Led_Off()` 用于定时器中断中与 `Led_Disp()` 交替调用，实现 PWM 亮度控制，没有它无法调光。  
- `Motor()` 和 `Relay()` 与 `Beep()` 共享同一 Y5 锁存器（同一个 `temp_2`），  
  比赛遇到继电器/电机题必须用到。

## 二、key.c

### 错误4：只扫描了一行按键，S8~S15 永远检测不到

**错误代码：**
```c
P44=0; P42=1; P35=1;
if(P30==0) temp=7;
if(P31==0) temp=6;
if(P32==0) temp=5;
if(P33==0) temp=4;
```

**正确代码（三行完整扫描 + UART 保护）：**
```c
// 第一行 S4~S7（P44 拉低）
P44=1; P42=1; P35=1;
if(P30==0) return 0;       // UART 冲突保护
P44=0; P42=1; P35=1;
if(P30==0) temp=7;
if(P31==0) temp=6;
if(P32==0) temp=5;
if(P33==0) temp=4;

// 第二行 S8~S11（P42 拉低）
P44=1; P42=1; P35=1;
if(P30==0) return 0;
P44=1; P42=0; P35=1;
if(P30==0) temp=11;
if(P31==0) temp=10;
if(P32==0) temp=9;
if(P33==0) temp=8;

// 第三行 S12~S15（P35 拉低）
P44=1; P42=1; P35=1;
if(P30==0) return 0;
P44=1; P42=1; P35=0;
if(P30==0) temp=15;
if(P31==0) temp=14;
if(P32==0) temp=13;
if(P33==0) temp=12;

// 恢复引脚状态
P44=1; P42=1; P35=1;
P3 = P3 | 0xef;
```

**原因：**  
CT107D 按键矩阵共三行，P44/P42/P35 分别控制一行。只写一行则 S8~S15 共 8 个按键永远不响应。

---

### 错误5：缺少 UART 冲突保护

**原因：**  
P3.0 既是串口 RXD，也是按键列线。串口接收数据时起始位拉低 P3.0，若不加  
`if(P30==0) return 0` 保护，会将串口数据误判为按键，导致按键乱报。

---

### 错误6：扫描结束后未恢复 P3.4

**原因：**  
P3.4 是 T0 外部计数输入引脚（频率测量用）。按键扫描拉低 P3.4 后若不恢复，  
会干扰频率测量。结束后必须加 `P3 = P3 | 0xef` 恢复高阻输入状态。

---

## 三、main.c

### 错误7：按键边沿检测逻辑写错

**错误代码：**
```c
Key_Val  = Key_Read();
Key_Up   = Key_Val & (Key_Val ^ Key_Up);   // 错：用了 Key_Up 自身
Key_Down = ~Key_Val & (Key_Val ^ Key_Up);  // 错：用了刚被改掉的 Key_Up
Key_Old  = Key_Val;
```

**正确代码：**
```c
Key_Val  = Key_Read();
Key_Down = ~Key_Val & (Key_Val ^ Key_Old); // 下降沿：刚被按下
Key_Up   =  Key_Val & (Key_Val ^ Key_Old); // 上升沿：刚被松开
Key_Old  = Key_Val;
```

**原因：**  
边沿检测核心是当前值与上一次值（Key_Old）异或，得到发生变化的位。  
错误代码把 `Key_Old` 误写成 `Key_Up`，导致两个标志位均计算错误，无法检测任何按键动作。  
注意：`Key_Down` 必须先算，`Key_Up` 后算，顺序不能颠倒。

## 四、ultrasound.c

### 错误8（第一次练习）：计时器在回波结束后才启动，永远返回错误值

**错误代码：**
```c
CMOD=0x00;
CH=CL=0;
CR=0;                          // 停止计时器
while((CF==0)&&(US_RX==1));    // 等待回波结束（ECHO 变低）
CR=1;                          // 回波结束后才开始计时！
```

**正确代码：**
```c
CMOD=0x00;
CH=CL=0;
Ut_Wave_Init();                // 发射脉冲（放入函数内部调用）
CR=1;                          // 先启动计时器
while((CF==0)&&(US_RX==1));    // 等待回波结束
CR=0;                          // 停止计时器
```

**原因：**  
超声波测距需要测量 ECHO 引脚保持高电平的持续时间。  
原代码在 ECHO 已经变低后才启动计时，CH/CL 始终接近 0，  
`0 × 0.017 + 2 ≈ 2`，无论真实距离多远，返回值永远是 2。  
同时，`Ut_Wave_Init()` 应放在 `Ut_Wave_Data()` 内部调用，  
确保发完脉冲立刻开始计时，避免错过回波。

### 错误9（第二次练习）：`=` 写成赋值而非比较，程序死循环

**错误代码：**
```c
while((CF==0)&&(US_RX=1));   // = 是赋值，将 US_RX 强制置 1，条件永远为真
```

**正确代码：**
```c
while((CF==0)&&(US_RX==1));  // == 才是比较
```

**原因：**  
`US_RX=1` 是赋值表达式，将 P1.1 强制拉高并返回值 1（非零即真）。  
整个循环条件永远成立，程序死循环卡死，无法返回任何距离值。  
**编译器不报错不报警**，这是 C 语言最隐蔽的常见错误之一，务必牢记。

---

### 错误10（第二次练习）：`time` 变量类型太小，高位数据全部丢失

**错误代码：**
```c
unsigned char time;        // char 只能存 0~255
time = CH<<8 | CL;         // 结果最大 65535，高位全部截断丢失
```

**正确代码：**
```c
unsigned int time;         // int 能存 0~65535
time = CH<<8 | CL;
```

**原因：**  
PCA 计数器 CH/CL 合并后是 16 位数值，最大 65535。  
`unsigned char` 只有 8 位，`CH<<8` 的高 8 位会被截断，  
导致距离计算结果完全错误（只保留低 8 位）。

## 五、iic.c

### 错误11：`Da_Write` 字节顺序错误，DAC 无法工作

**错误代码：**
```c
I2CSendByte(dat);    // dat 被当作控制字节发出
I2CWaitAck();
I2CSendByte(0x91);   // 0x91 是读地址，被当作 DAC 数字量发出
I2CWaitAck();
```

**正确代码：**
```c
I2CSendByte(0x41);   // 控制字节：bit6=1 使能 DA，bit[1:0]=01 选通道1
I2CWaitAck();
I2CSendByte(dat);    // 真正的 DAC 数字量（0~255 对应 0~5V）
I2CWaitAck();
```

**原因：**  
PCF8591 的 DAC 写时序：设备地址(0x90) → 控制字节(0x41) → DAC 值。  
原代码把 `dat` 和 `0x91` 的位置对调，DAC 输出完全错误。

---

### 错误12：`EEPROM_Read` 的 `I2CStop()` 写在循环内部（两次练习均犯）

**错误代码：**
```c
while(num--)
{
    *str++ = I2CReceiveByte();
    if(num) I2CSendAck(0);
    else    I2CSendAck(1);
    I2CStop();    // ← 每读一个字节就停止总线！
}
```

**正确代码：**
```c
while(num--)
{
    *str++ = I2CReceiveByte();
    if(num) I2CSendAck(0);
    else    I2CSendAck(1);
}
I2CStop();        // ← 全部读完后再停止
```

**原因：**  
I2C 连续读操作：每个非末字节发 ACK(0) 表示"继续发"，末字节发 NACK(1) 后才发 STOP。  
把 `I2CStop()` 放循环里，读完第一个字节就终止总线，后续字节全部读取失败。  
**⚠ 这个错误两次练习都犯了，重点记忆：I2CStop() 必须在循环外面。**

---

### 错误13：`EEPROM_Read` 切换读模式后缺少 `I2CWaitAck()`

**错误代码：**
```c
I2CStart();
I2CSendByte(0xa1);    // 发送读地址
                      // 缺少 I2CWaitAck()
while(num--)
```

**正确代码：**
```c
I2CStart();
I2CSendByte(0xa1);
I2CWaitAck();         // 等待从机应答
while(num--)
```

**原因：**  
I2C 协议规定每发一个字节必须等从机 ACK 后再继续。  
漏掉 `I2CWaitAck()` 导致 SCL 时序错位，从机数据读取失败。

---

### 错误14：`I2C_Delay` 每次循环只有 1 个 `_nop_()`，I2C 时序太快

**错误代码：**
```c
static void I2C_Delay(unsigned char n)
{
    do { _nop_(); } while(n--);
}
```

**正确代码：**
```c
static void I2C_Delay(unsigned char n)
{
    do
    {
        _nop_();_nop_();_nop_();_nop_();_nop_();
        _nop_();_nop_();_nop_();_nop_();_nop_();
        _nop_();_nop_();_nop_();_nop_();_nop_();
    }
    while(n--);
}
```

**原因：**  
I2C 标准模式要求信号保持时间至少 4µs。1 个 nop 在 12MHz 下约 0.08µs，  
`DELAY_TIME=5` 时总延时仅约 0.4µs，远不够，导致 ADC/EEPROM 读写失败。  
每次循环 15 个 nop 约 6µs，满足时序要求。

---

### 错误15：`EEPROM_Write` 写完后等待时间不足

**错误代码：**
```c
I2CStop();
I2C_Delay(250);  // 3次
I2C_Delay(250);
I2C_Delay(250);
```

**正确代码：**
```c
I2CStop();
I2C_Delay(255);  // 10次
I2C_Delay(255);
I2C_Delay(255);
I2C_Delay(255);
I2C_Delay(255);
I2C_Delay(255);
I2C_Delay(255);
I2C_Delay(255);
I2C_Delay(255);
I2C_Delay(255);
```

**原因：**  
AT24C02 每次字节写操作内部需要约 5ms 完成。  
等待不足时下次操作发起时 EEPROM 还未写完，数据会丢失或出错。

## 六、ds1302.c

### 错误16：`2i` 非法语法，编译直接失败

**错误代码：**
```c
Write_Ds1302_Byte( 0x84-2i , ucRtc[i]/10*16+ucRtc[i]%10 )    // 无分号，2i 不合法
temp=Read_Ds1302_Byte ( 0x85-2i )                              // 无分号，2i 不合法
```

**正确代码：**
```c
Write_Ds1302_Byte(0x84 - 2*i, ucRtc[i]/10*16 + ucRtc[i]%10);
temp = Read_Ds1302_Byte(0x85 - 2*i);
```

**原因：**  
C 语言中 `2i` 不是合法表达式（数字后直接跟变量名），编译器直接报错，无法生成 hex。  
正确写法是 `2*i`，同时两行末尾均漏写了分号 `;`。

---

### 错误17：BCD 解码公式错误（运算符优先级陷阱）

**错误代码：**
```c
ucRtc[i] = temp/16*10 + temp&16;
```

**正确代码：**
```c
ucRtc[i] = temp/16*10 + temp%16;
```

**原因：**  
DS1302 读出的是 BCD 码，解码公式为：十进制 = 高4位×10 + 低4位。  
`temp%16` 正确取低4位（0~9）。  
`temp&16` 实际是 `temp&0x10`，只检测 bit4，结果只有 0 或 16，完全错误。  

更隐蔽的是运算符优先级：`&` 优先级低于 `+`，所以  
`temp/16*10 + temp&16` 被解析为 `(temp/16*10 + temp) & 16`，结果完全错误。  
遇到位运算和算术混用时，**不确定就加括号**。

## 总结：高频/易错点速查

| 编号 | 错误类型 | 出现次数 | 核心记忆点 |
|------|----------|----------|------------|
| 1 | `EEPROM_Read` 的 `I2CStop()` 在循环里 | **2次** | Stop 必须在循环外面 |
| 2 | `=` 写成赋值而非 `==` 比较 | 1次 | 编译不报错，程序死循环 |
| 3 | 超声波计时器启动时机错误 | 1次 | CR=1 必须在 while 之前 |
| 4 | `unsigned char` 存 16位值 | 1次 | 16位变量必须用 `unsigned int` |
| 5 | 运算符优先级 `&` 低于 `+` | 1次 | 混用时加括号 |
| 6 | `2i` 忘写乘号 `*` | 1次 | 数字×变量必须写 `2*i` |
| 7 | 按键只扫一行 | 1次 | 三行：P44 / P42 / P35 |
| 8 | 缺 UART 冲突保护 | 1次 | 每行扫描前先判断 P30 |
| 9 | `temp_1_old` 初始值设错 | 1次 | 初始值要与硬件初始状态一致 |
| 10 | `Da_Write` 字节顺序错 | 1次 | 先控制字节 0x41，再 DAC 值 |
