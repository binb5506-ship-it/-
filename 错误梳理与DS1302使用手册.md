# 错误梳理与 DS1302 使用手册

---

## 一、错误代码对比与解释

---

### 错误1：`Scheduler_Init` 漏括号

**错误代码**
```c
void main()
{
    Scheduler_Init;     // 这只是引用函数地址，函数根本没有被调用！
    Uart1_Init();
    ...
}
```

**正确代码**
```c
void main()
{
    Scheduler_Init();   // 加括号才是真正调用函数
    Uart1_Init();
    ...
}
```

**解释**  
在 C 语言中，`函数名` 不加括号只是取函数的地址（一个指针），不会执行函数体。
`函数名()` 加括号才是调用。这个错误不报编译错误，只有警告，极其隐蔽。

---

### 错误2：`main()` 缺少 `System_Init()` 和 `Timer1_Init()`

**错误代码**
```c
void main()
{
    Scheduler_Init;     // 还漏了括号
    Uart1_Init();
    while(1) { Scheduler_Run(); }
    // 缺 System_Init() → 端口初始态混乱，LED/数码管上电乱显示
    // 缺 Timer1_Init() → 定时器不运行，uwTick永远为0，调度器和串口超时全部失效
}
```

**正确代码**
```c
void main()
{
    System_Init();      // 第一步：初始化端口状态（P0/P2），防止上电乱显
    Uart1_Init();       // 初始化串口
    Scheduler_Init();   // 初始化调度器任务数量
    Timer1_Init();      // 启动定时器，产生1ms时基，调度器和串口才能正常工作
    while(1)
    {
        Scheduler_Run();
    }
}
```

**解释**  
- `System_Init()`：对 P0/P2 端口做初始化，防止上电时 LED 和数码管显示乱码。
- `Timer1_Init()`：启动 Timer1，每 1ms 进一次中断，`uwTick++`。
  没有它，`uwTick` 永远是 0，调度器里 `now_time >= last_ms + rate_ms` 永远成立，
  所有任务会在每次 `Scheduler_Run()` 都被调用，完全乱套。
  同时 `Uart_Rx_Tick` 也无法累加，串口超时判断永久失效。

---

### 错误3：`Scheduler_Run()` 没有 for 循环 + `i` 未初始化 + 用 `>` 而非 `>=`

**错误代码**
```c
void Scheduler_Run()
{
    idata unsigned char i;                      // i 是垃圾值，从未赋值
    idata unsigned long int now_time = uwTick;
    if(now_time > Scheduler_Task[i].last_ms + Scheduler_Task[i].rate_ms)
    {                                           // 没有循环，只跑一次，且 i 不确定
        Scheduler_Task[i].last_ms = now_time;
        Scheduler_Task[i].task_func();
    }
}
```

**正确代码**
```c
void Scheduler_Run()
{
    unsigned char i;
    for(i = 0; i < task_num; i++)               // for 循环遍历所有任务
    {
        unsigned long int now_time = uwTick;    // 每个任务单独取当前时间
        if(now_time >= Scheduler_Task[i].last_ms + Scheduler_Task[i].rate_ms)
        {                                       // >= 而不是 >
            Scheduler_Task[i].last_ms = now_time;
            Scheduler_Task[i].task_func();
        }
    }
}
```

**解释**  
- 没有 `for` 循环：只有第一个任务（甚至是随机内存位置）会被执行，其余任务永远不跑。
- `i` 未初始化：`idata unsigned char i;` 未赋初值，i 是内存里的随机垃圾，
  访问 `Scheduler_Task[i]` 会读取错误位置的内存，导致程序崩溃或乱跑。
- `>` vs `>=`：用 `>` 会让每个任务晚执行 1ms，用 `>=` 才在恰好到期时执行。

---

### 错误4：`Scheduler_Task[]` 是空的

**错误代码**
```c
idata task_t Scheduler_Task[] =
{
    // 空的！没有注册任何任务
};
```

**正确代码**
```c
idata task_t Scheduler_Task[] =
{
    {Led_Proc,  1,   0},    // LED刷新，每1ms
    {Key_Proc,  10,  0},    // 按键扫描，每10ms
    {Seg_Proc,  20,  0},    // 数码管逻辑，每20ms
    {Uart_Proc, 10,  0},    // 串口处理，每10ms
    // 根据题目需要添加其他任务
};
```

**解释**  
调度器靠 `sizeof(Scheduler_Task)/sizeof(task_t)` 计算任务数量。
数组为空时 `task_num = 0`，`for` 循环一次都不执行，程序什么都不干。
每道题都要根据用到的模块往这里注册对应的处理函数。

---

### 错误5：UART 中断里 `Ua_Rx_Index` 没有自增 + 溢出判断错误

**错误代码**
```c
void Uart1_Isr(void) interrupt 4
{
    if(RI)
    {
        Ua_Rx_Flag = 1;
        Ua_Rx_Tick = 0;
        Ua_Rx_Buf[Ua_Rx_Index] = SBUF;  // 存进去，但 Index 没有加！
        RI = 0;
        if(Ua_Rx_Index > 10)            // 判断在自增前，永远判断的是旧值
        {                               // 且 > 10 应为 >= 10，否则会写 buf[10] 越界
            Ua_Rx_Index = 0;
            memset(Ua_Rx_Buf, 0, 10);
        }
    }
}
```

**正确代码**
```c
void Uart1_Isr(void) interrupt 4
{
    if(RI)
    {
        Ua_Rx_Flag = 1;
        Ua_Rx_Tick = 0;
        Ua_Rx_Buf[Ua_Rx_Index++] = SBUF;  // 存入后立刻自增
        RI = 0;
        if(Ua_Rx_Index >= 10)              // 自增后判断，用 >= 防止越界
        {
            Ua_Rx_Index = 0;
            memset(Ua_Rx_Buf, 0, 10);
        }
    }
}
```

**解释**  
- 没有 `++`：`Ua_Rx_Index` 永远是 0，每次收到的字节都覆盖 `Ua_Rx_Buf[0]`，
  接收缓冲区只有一个字节有效，`strcmp` 等命令解析永远失败。
- `> 10` vs `>= 10`：缓冲区是 `[10]`，合法下标是 0~9。
  `Ua_Rx_Index` 自增到 10 时，下一次存储就会写 `buf[10]`，越界访问内存。
  应该在 Index 达到 10 时就重置，即用 `>= 10`。

## 二、完整 DS1302 时钟代码

---

### ds1302.h

```c
#include <STC15F2K60S2.H>

void Set_Rtc(unsigned char *ucRtc);
void Read_Rtc(unsigned char *ucRtc);
```

---

### ds1302.c

```c
#include "ds1302.h"
#include "intrins.h"

// 引脚定义（根据实际硬件连接修改）
sbit SCK = P1^7;   // 时钟线
sbit SDA = P2^3;   // 数据线
sbit RST = P1^3;   // 片选/复位线

// 向 DS1302 写一个字节（低位先发）
void Write_Ds1302(unsigned char temp)
{
    unsigned char i;
    for(i = 0; i < 8; i++)
    {
        SCK = 0;
        SDA = temp & 0x01;  // 取最低位
        temp >>= 1;
        SCK = 1;
    }
}

// 向指定地址写一个字节
void Write_Ds1302_Byte(unsigned char address, unsigned char dat)
{
    RST = 0; _nop_();
    SCK = 0; _nop_();
    RST = 1; _nop_();
    Write_Ds1302(address);
    Write_Ds1302(dat);
    RST = 0;
}

// 从指定地址读一个字节
unsigned char Read_Ds1302_Byte(unsigned char address)
{
    unsigned char i, temp = 0x00;
    RST = 0; _nop_();
    SCK = 0; _nop_();
    RST = 1; _nop_();
    Write_Ds1302(address);
    for(i = 0; i < 8; i++)
    {
        SCK = 0;
        temp >>= 1;
        if(SDA) temp |= 0x80;
        SCK = 1;
    }
    RST = 0; _nop_();
    SCK = 0; _nop_();
    SCK = 1; _nop_();
    SDA = 0; _nop_();
    SDA = 1; _nop_();
    return temp;
}

/**
 * 设置时间
 * ucRtc[0]=时, ucRtc[1]=分, ucRtc[2]=秒
 * 例：设置 23:59:50 → ucRtc = {23, 59, 50}
 */
void Set_Rtc(unsigned char *ucRtc)
{
    unsigned char i;
    Write_Ds1302_Byte(0x8E, 0x00);  // 关闭写保护
    Write_Ds1302_Byte(0x80, 0x80);  // 暂停时钟（CH位置1）
    for(i = 0; i < 3; i++)
    {
        // 地址：0x84(时) 0x82(分) 0x80(秒)，十进制转BCD写入
        Write_Ds1302_Byte(0x84 - 2 * i, ucRtc[i] / 10 * 16 + ucRtc[i] % 10);
    }
    Write_Ds1302_Byte(0x8E, 0x80);  // 开启写保护
}

/**
 * 读取时间
 * ucRtc[0]=时, ucRtc[1]=分, ucRtc[2]=秒
 * 读出的是十进制，可直接使用
 */
void Read_Rtc(unsigned char *ucRtc)
{
    unsigned char i, temp;
    EA = 0;  // 关中断，防止读取过程被打断导致数据错误
    for(i = 0; i < 3; i++)
    {
        // 地址：0x85(时) 0x83(分) 0x81(秒)，读出BCD转十进制
        temp = Read_Ds1302_Byte(0x85 - 2 * i);
        ucRtc[i] = temp / 16 * 10 + temp % 16;
    }
    EA = 1;
}
```

## 三、DS1302 使用方法

### 第一步：声明时间数组和相关变量

```c
// 在 main.c 全局变量区声明
pdata unsigned char ucRtc[3] = {11, 12, 13};  // 初始时间 11:12:13
                                               // [0]=时 [1]=分 [2]=秒
```

---

### 第二步：main() 里设置初始时间

```c
void main()
{
    System_Init();
    Set_Rtc(ucRtc);     // 上电时写入初始时间，只需调用一次
    Scheduler_Init();
    Timer1_Init();
    while(1) { Scheduler_Run(); }
}
```

---

### 第三步：添加读时间任务

```c
void Get_Time()
{
    Read_Rtc(ucRtc);    // 将当前时间读入 ucRtc 数组
}

// 调度器里注册，每100ms读一次
idata task_t Scheduler_Task[] =
{
    {Led_Proc,  1,   0},
    {Key_Proc,  10,  0},
    {Seg_Proc,  20,  0},
    {Get_Time,  100, 0},   // ← 加这一行
};
```

---

### 第四步：数码管显示时间（Seg_Proc 里）

```c
void Seg_Proc()
{
    // 显示格式：HH-MM-SS
    Seg_Buf[0] = ucRtc[0] / 10;        // 时 十位
    Seg_Buf[1] = ucRtc[0] % 10;        // 时 个位
    Seg_Buf[2] = 0 + ',';              // 显示 '-' （加','偏移触发小数点位）
    Seg_Buf[3] = ucRtc[1] / 10;        // 分 十位
    Seg_Buf[4] = ucRtc[1] % 10;        // 分 个位
    Seg_Buf[5] = 0 + ',';              // 显示 '-'
    Seg_Buf[6] = ucRtc[2] / 10;        // 秒 十位
    Seg_Buf[7] = ucRtc[2] % 10;        // 秒 个位
}
```

---

### 第五步：按键调整时间（Key_Proc 里典型写法）

```c
// 全局变量
idata unsigned char ucRtc_Index = 0;  // 0=时 1=分 2=秒
idata bit Time_Set = 0;               // 0=正常显示 1=调整模式

void Key_Proc()
{
    Key_Val = Key_Read();
    Key_Down = Key_Val & (Key_Val ^ Key_Old);
    Key_Up   = ~Key_Val & (Key_Val ^ Key_Old);
    Key_Old  = Key_Val;

    // S7：进入/切换调整位
    if(Key_Down == 7)
    {
        if(Time_Set == 0)
        {
            Time_Set = 1;
            ucRtc_Index = 0;
        }
        else
        {
            ucRtc_Index = (++ucRtc_Index) % 3;
            if(ucRtc_Index == 0)
            {
                Time_Set = 0;
                Set_Rtc(ucRtc);    // 调整完毕，写回 DS1302
            }
        }
    }

    // S6：加一
    if(Key_Down == 6 && Time_Set == 1)
    {
        ucRtc[ucRtc_Index]++;
        if(ucRtc_Index == 0 && ucRtc[0] >= 24) ucRtc[0] = 0;
        if(ucRtc_Index == 1 && ucRtc[1] >= 60) ucRtc[1] = 0;
        if(ucRtc_Index == 2 && ucRtc[2] >= 60) ucRtc[2] = 0;
    }

    // S5：减一
    if(Key_Down == 5 && Time_Set == 1)
    {
        if(ucRtc[ucRtc_Index] == 0)
        {
            if(ucRtc_Index == 0) ucRtc[0] = 23;
            else ucRtc[ucRtc_Index] = 59;
        }
        else ucRtc[ucRtc_Index]--;
    }
}
```

---

### BCD 码说明

DS1302 内部存储的是 **BCD 码**，不是普通十进制。

| 实际时间 | 十进制 | BCD码（存入DS1302） |
|---------|--------|-------------------|
| 23时    | 23     | 0x23（高4位=2，低4位=3）|
| 59分    | 59     | 0x59              |
| 50秒    | 50     | 0x50              |

- **写入**：`十进制 / 10 * 16 + 十进制 % 10` → 转BCD
- **读出**：`BCD / 16 * 10 + BCD % 16` → 转十进制

ds1302.c 里已封装好，**直接用 `Set_Rtc` 和 `Read_Rtc` 传十进制即可，无需手动转换**。
