# 闪烁

> 对于闪烁，我们定义一个时间位，然后还需要定义一个标志位 ，然后我们需要注意，当我们不使用闪烁（在这里就是不处于设置界面的时候），我们需要把闪烁的500ms给清零，避免我们后续闪烁的时间不准确，灵活运用三目操作符，可以让我们快速控制闪烁。

```c
idata unsigned int time_500ms = 0;	// 500ms计时
idata bit Setting_Mode = 0;			// 0 未设置 1 设置
idata bit Light_Seg_Mode = 0;		// 闪烁标志位数码管

```

```C
if (Setting_Mode)
{
    // 处于设置界面
    if (++time_500ms == 500)
    {
        Light_Seg_Mode ^= 1;
        time_500ms = 0;
    }
}
else
{
    // 不处于设置界面
    time_500ms = 0;
    Light_Seg_Mode = 0;
}
```

## 赋值

> 赋值的时候，我们要注意先`--`和后`--`的区别，一个是运算后赋值，另一个是先赋值后运算 

假设你是一个学生，正在进行课程报名。你可以选择先报名，然后才查看是否报名成功，或者先查看名额是否充足，再进行报名。

1. **先报名后查看 (`--a`)**： 你决定先报名一门课程，然后再查看课程的剩余名额。在报名之后，你会立刻知道自己已经减少了一个名额。

   就像在 C 语言中 `--a`，你会先将 `a` 减去1（即先报名），然后再查看或使用这个值。

2. **先查看后报名 (`a--`)**： 你先查看课程剩余名额，确保可以报名。然后你才决定报名一门课程，减少一个名额。

   在 C 语言中 `a--` 就是你先使用当前的值（比如查看剩余名额），然后才把 `a` 减去1（即报名后减少名额）。

总结：

- **`--a`**：先改变，再使用（先报名，再查看名额）。
- **`a--`**：先使用，再改变（先查看名额，再报名）。
